# Next.js + shadcn Repo Initialization Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Initialize this repository as a working Next.js App Router project in the repo root with TypeScript, Tailwind CSS, and shadcn UI, while preserving and extending the existing README.

**Architecture:** Create a single frontend application at the repository root, keep the first pass static and component-driven, and wire in shadcn primitives plus a minimal branded home page that proves the foundation works. Treat the initial scope as framework setup, not product-feature development.

**Tech Stack:** Next.js App Router, React 19, TypeScript, Tailwind CSS, ESLint, shadcn UI, npm

---

**Execution notes:**

- Do not create a branch, worktree, or commit unless the user explicitly asks.
- TDD is not practical for framework-generated bootstrap files; use build and lint verification as the quality gate for scaffold tasks.
- Preserve the current `README.md` content and append setup guidance instead of replacing it.

## File Map

### Files to create

- `.gitignore` - ignore Next.js build outputs, dependencies, and local environment files.
- `package.json` - project metadata, scripts, and dependencies.
- `package-lock.json` - npm lockfile generated by install.
- `next-env.d.ts` - Next.js TypeScript environment declarations.
- `tsconfig.json` - TypeScript config with `@/*` alias support.
- `next.config.ts` - Next.js config placeholder if scaffold provides one.
- `eslint.config.mjs` - explicit ESLint config for Next.js.
- `postcss.config.mjs` - PostCSS/Tailwind integration.
- `components.json` - shadcn configuration metadata.
- `app/layout.tsx` - root document shell, metadata, and font wiring.
- `app/page.tsx` - starter homepage using shadcn components.
- `app/globals.css` - Tailwind base styles, theme tokens, and page styling.
- `app/favicon.ico` - default app icon from the scaffold.
- `components/ui/button.tsx` - shadcn button primitive.
- `components/ui/card.tsx` - shadcn card primitive.
- `lib/utils.ts` - utility helpers such as `cn`.
- `public/*` - any default public assets copied from scaffold if present.

### Files to modify

- `README.md` - preserve the existing product brief and append setup commands plus a short initialized-scope section.
